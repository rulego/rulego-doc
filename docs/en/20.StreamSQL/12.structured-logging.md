# Structured Logging

The `github.com/rulego/streamsql/logger` package adds a **structured logging** layer on top of the existing printf-style logger: it carries context as key-value pairs and supports both logfmt text and JSON output.

## Overview

Printf logs are friendly to humans but hostile to log aggregators (ELK, Loki, Datadog, etc.) ŌĆö extracting fields from `"window fired for device d1 in 5ms"` means writing regexes. Structured logging splits context into independent key-value pairs so aggregators can index and filter directly on `device=d1` or `duration=5ms`.

The layer is **additive**:

- The printf signatures of `Debug`/`Info`/`Warn`/`Error` are **completely unchanged**, so existing custom `Logger` implementations and call sites keep working.
- Four new methods `DebugFields`/`InfoFields`/`WarnFields`/`ErrorFields` accept `...Field`.
- A new constructor `NewLoggerWithFormat` selects the output encoding.

::: tip No breakage
Your existing `logger.Logger` implementation (the four printf methods only) needs no changes at all. You only implement the `StructuredLogger` interface when you want to emit structured key-value output.
:::

## Field Constructors

```go
type Field struct {
    Key   string
    Value interface{}
}
```

| Constructor | Description |
|-------------|-------------|
| `logger.String(k, v)` | string value |
| `logger.Int(k, v)` | `int` value |
| `logger.Int64(k, v)` | `int64` value |
| `logger.Float64(k, v)` | `float64` value |
| `logger.Bool(k, v)` | `bool` value |
| `logger.Err(err)` | error value, always written under the key `"error"`; a `nil` error is preserved as null and does not panic |
| `logger.Duration(k, d)` | `time.Duration`, recorded as a human-readable string such as `"12ms"` |
| `logger.Any(k, v)` | arbitrary value; prefer a typed constructor when one exists |

::: tip Why typed constructors
`Field.Value` is `interface{}`. Passing `Field{Key: "n", Value: 42}` directly boxes `42` as an `int`, but reading it back in a context that expects `int64` fails the type assertion. Typed constructors pin the type at the call site and avoid boxing surprises.
:::

## Two Output Formats

```go
type Format int

const (
    TextFormat Format = iota // "[ts] [LEVEL] msg key=value", logfmt-style. Default
    JSONFormat               // one JSON object per line
)
```

The same call rendered both ways:

```go
logger.InfoFields("window fired",
    logger.String("device", "d1"),
    logger.Int("count", 42),
    logger.Duration("duration", 5200*time.Microsecond),
)
```

Text (default):

```
[2026-07-08 10:00:00.123] [INFO] window fired device=d1 count=42 duration=5.2ms
```

JSON:

```
{"ts":"2026-07-08 10:00:00.123","level":"INFO","msg":"window fired","device":"d1","count":42,"duration":"5.2ms"}
```

Both formats carry three fixed fields ŌĆö `ts` (timestamp, millisecond precision), `level`, `msg` ŌĆö followed by the caller-supplied fields.

## Creating a Structured Logger

```go
func NewLoggerWithFormat(level Level, output io.Writer, format Format) Logger
```

```go
import (
    "os"

    "github.com/rulego/streamsql/logger"
)

// JSON format, INFO level, output to stdout
log := logger.NewLoggerWithFormat(logger.INFO, os.Stdout, logger.JSONFormat)
```

Plug it into StreamSQL so all internal logs go through it:

```go
logger.SetDefault(logger.NewLoggerWithFormat(logger.INFO, os.Stdout, logger.JSONFormat))

ssql := streamsql.New() // internal logs use the JSON logger above
```

::: tip Relation with Options
`streamsql.WithLogLevel(level)` and `streamsql.WithDiscardLog()` still work ŌĆö the structured layer is additive and does not break their existing behavior. To fully replace the output format, install a structured logger via `SetDefault`.
:::

Level constants match the printf logger: `DEBUG`, `INFO` (default), `WARN`, `ERROR`, `OFF`.

## The StructuredLogger Interface

A custom logger implements this interface when it wants to emit structured fields:

```go
type StructuredLogger interface {
    DebugFields(msg string, fields ...Field)
    InfoFields(msg string, fields ...Field)
    WarnFields(msg string, fields ...Field)
    ErrorFields(msg string, fields ...Field)
}
```

The built-in `defaultLogger` (returned by `NewLogger` / `NewLoggerWithFormat`) implements **both** `Logger` and `StructuredLogger`, so the printf and structured paths share the same rendering logic. `discardLogger` also implements `StructuredLogger` (all four methods are no-ops).

## Package-level Helpers

```go
func InfoFields(msg string, fields ...Field)
func WarnFields(msg string, fields ...Field)
func ErrorFields(msg string, fields ...Field)
func DebugFields(msg string, fields ...Field)
```

These four functions operate on the **global default logger** (the one set by `SetDefault`, defaulting to `NewLogger(INFO, os.Stdout)`):

```go
logger.InfoFields("emit", logger.String("device", "d1"), logger.Int("bytes", 128))
```

::: warning Fallback for printf-only loggers
If the default logger does **not** implement `StructuredLogger` (for example a custom `Logger` that only implements the four printf methods), the `InfoFields` family of helpers fold all fields into a `"key=value key=value"` text snippet appended to the message and make a single printf call. **No context is lost**, but there is no JSON output either.
:::

The printf-style package helpers (`logger.Info`, `logger.Warn`, `logger.Error`, `logger.Debug`), the constructors (`NewLogger`, `NewDiscardLogger`), the accessors (`SetDefault`, `GetDefault`), and the `Logger` interface itself (`Debug`/`Info`/`Warn`/`Error`/`SetLevel`) all remain unchanged.

## Complete Example

```go
package main

import (
    "fmt"
    "os"
    "time"

    "github.com/rulego/streamsql"
    "github.com/rulego/streamsql/logger"
)

func main() {
    // 1. Install a JSON structured logger as the global default
    logger.SetDefault(logger.NewLoggerWithFormat(logger.INFO, os.Stdout, logger.JSONFormat))

    ssql := streamsql.New()
    defer ssql.Stop()

    ssql.Execute(`SELECT deviceId, AVG(temperature) AS avg_t
                  FROM stream
                  GROUP BY deviceId, TumblingWindow('5s')`)
    ssql.AddSink(func(rows []map[string]interface{}) {
        if len(rows) > 0 {
            // Emit a structured log line from the sink
            logger.InfoFields("window emitted",
                logger.String("device", fmt.Sprintf("%v", rows[0]["deviceId"])),
                logger.Any("avg_t", rows[0]["avg_t"]),
            )
        }
    })

    for i := 0; i < 3; i++ {
        ssql.Emit(map[string]interface{}{
            "deviceId":    "d1",
            "temperature": 20.0 + float64(i),
        })
    }

    time.Sleep(6 * time.Second)
}
```

After running, stdout shows lines like (StreamSQL's internal window/scheduler logs use the same format):

```
{"ts":"2026-07-08 10:00:05.123","level":"INFO","msg":"window emitted","device":"d1","avg_t":21}
```
