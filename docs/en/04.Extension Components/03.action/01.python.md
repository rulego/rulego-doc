The `x/python` component is a Python script action node. It executes Python scripts via subprocess to transform, filter, or enrich messages. Supports both inline script and external `.py` file modes.

## Prerequisites

- Python 3.x must be installed and `python3` or `python` must be available in the system PATH
- All Python 3 standard libraries and installed third-party packages are supported

## Configuration

| Field | Type | Description | Default |
|-------|------|-------------|---------|
| script | string | Python script content (inline mode) or `.py` file path (file mode) | None (required) |
| pythonPath | string | Python executable path | Auto-detect (tries `python3`, then `python`) |
| timeout | string | Script execution timeout. Supports Go duration format (`"5s"`, `"1000ms"`, `"1m"`) or plain number in seconds (`"5"`) | Uses rule engine global config `ScriptMaxExecutionTime` |
| maxRunning | int | Maximum number of concurrent Python subprocesses | 10 |

### script Field

**Inline mode**: Provide only the function body. Do not include the `def Process(...):` line.

```python
# Inline mode: function body only
import json
data = json.loads(msg) if isinstance(msg, str) else msg
data["processed"] = True
return json.dumps(data), metadata, msgType
```

**File mode**: A file path ending with `.py`. The file must define a `Process` function.

```python
# File mode: complete function definition
import json

def Process(msg, metadata, msgType, dataType):
    data = msg if isinstance(msg, dict) else json.loads(msg)
    data["processed"] = True
    return json.dumps(data), metadata, msgType
```

::: warning Inline mode restriction
Do not write `def Process(...):` in inline mode, or an initialization error will occur. Defining helper functions (e.g., `def helper(x):`) is allowed.
:::

## Function Parameters

The `Process` function supports 4 to 6 parameters. The framework uses `inspect.signature` to adapt automatically:

| Parameter | Type | Description | Required |
|-----------|------|-------------|----------|
| msg | dict / str / list | Message data. If dataType is JSON, it is automatically parsed as a Python dict; otherwise it is a string | Yes |
| metadata | dict | Message metadata, a `str -> str` dictionary | Yes |
| msgType | str | Message type | Yes |
| dataType | str | Data type (e.g., `"JSON"`, `"TEXT"`, `"BINARY"`) | Yes |
| vars | dict | Node configuration variables, a `str -> str` dictionary | No |
| globalProps | dict | Global properties from rule engine config, a `str -> str` dictionary | No |

### Parameter Adaptation

```python
# 4 parameters: standard signature
def Process(msg, metadata, msgType, dataType):
    ...

# 5 parameters: additionally receives vars
def Process(msg, metadata, msgType, dataType, vars={}):
    server = vars.get("server", "unknown")

# 6 parameters: additionally receives vars and globalProps
def Process(msg, metadata, msgType, dataType, vars={}, globalProps={}):
    env = globalProps.get("env", "unknown")
```

## Return Values

The `Process` function supports 3 return forms:

### 1. Tuple

Return up to 3 elements corresponding to `msg`, `metadata`, and `msgType`:

```python
return new_msg, metadata, "NEW_TYPE"    # full 3-tuple
return new_msg, metadata                 # modify msg and metadata only
return new_msg                           # modify msg only
```

### 2. Dict

Return a dictionary containing `msg`, `metadata`, or `msgType` keys:

```python
return {"msg": "transformed", "msgType": "PROCESSED"}
return {"msg": new_data, "metadata": metadata, "msgType": msgType}
```

::: tip Dict recognition rule
A returned dict is treated as a structured result only if it contains at least one of the keys `msg`, `metadata`, or `msgType`. Otherwise, it is wrapped as `{"msg": result}`.
:::

### 3. Single Value

Return any value, automatically wrapped as `{"msg": value}`:

```python
return "plain string result"
return 42
return [1, 2, 3]
return {"data": 123}  # no msg/metadata/msgType keys, entire dict becomes msg
```

## Error Handling

When a script raises an exception, the node sends the message to the `Failure` chain. The error message includes the Python traceback:

```python
# Raise an exception in the script
if temperature > 100:
    raise ValueError(f"Temperature exceeded: {temperature}")
```

The node delivers the error through `TellFailure`:

```go
ctx.TellFailure(msg, err)  // err contains the Python traceback
```

::: warning Common error scenarios
- **Syntax error**: Script code does not conform to Python syntax
- **Runtime error**: KeyError, ValueError, TypeError, etc.
- **Timeout**: Script execution exceeds the configured timeout
- **Python not installed**: System cannot find the python executable
:::

## Relation Type

- ***Success:*** Script executed successfully, transformed message is sent to the `Success` chain
- ***Failure:*** Script execution failed (exception, timeout, syntax error, etc.), original message is sent to the `Failure` chain

## Configuration Examples

### Basic Message Transformation

```json
{
  "id": "s1",
  "type": "x/python",
  "name": "Temperature Conversion",
  "configuration": {
    "script": "import json\ndata = json.loads(msg) if isinstance(msg, str) else msg\ndata[\"temperature\"] = data[\"temperature\"] * 1.8 + 32\ndata[\"unit\"] = \"F\"\nreturn json.dumps(data), metadata, msgType"
  }
}
```

### Modify Metadata

```json
{
  "id": "s2",
  "type": "x/python",
  "name": "Add Metadata",
  "configuration": {
    "script": "metadata[\"processedAt\"] = \"2026-04-23\"\nmetadata[\"processor\"] = \"python\"\nreturn msg, metadata, msgType"
  }
}
```

### Change Message Type

```json
{
  "id": "s3",
  "type": "x/python",
  "name": "Change Message Type",
  "configuration": {
    "script": "return msg, metadata, \"ALARM\""
  }

```

### Using vars Configuration Variables

```json
{
  "id": "s4",
  "type": "x/python",
  "name": "Use Config Variables",
  "configuration": {
    "script": "metadata[\"server\"] = vars.get(\"server\", \"unknown\")\nmetadata[\"env\"] = vars.get(\"env\", \"unknown\")\nreturn msg, metadata, msgType",
    "vars": {
      "server": "prod-server-01",
      "env": "production"
    }
  }
}
```

::: tip vars vs globalProps
- **vars**: Node-level configuration variables set in the node configuration. Suitable for server addresses, thresholds, etc.
- **globalProps**: Global rule engine properties set via `config.Properties`. Suitable for environment variables, shared configuration, etc.
:::

### Using globalProps

```go
config := types.NewConfig()
config.Properties = types.Properties{

    "region": "us-east-1",

```

```json
{

    "script": "import json\ndata = json.loads(msg) if isinstance(msg, str) else msg\ndata[\"region\"] = globalProps.get(\"region\", \"unknown\")\nreturn json.dumps(data), metadata, msgType"

```

### External Python File

```json
{
  "id": "s6",
  "type": "x/python",
  "name": "External Script",
  "configuration": {
    "script": "/path/to/process.py",
    "pythonPath": "python3",
    "timeout": "10s",
    "maxRunning": 5
  }
}
```

Corresponding `process.py` file:

```python
import json

def Process(msg, metadata, msgType, dataType, vars={}, globalProps={}):
    data = msg if isinstance(msg, dict) else json.loads(msg)
    data["server"] = vars.get("server", "unknown")
    data["env"] = globalProps.get("env", "unknown")
    metadata["processedBy"] = "python"
    return json.dumps(data), metadata, msgType
```

### Using Third-Party Libraries

```json
{
  "id": "s7",
  "type": "x/python",
  "name": "HTTP Request",
  "configuration": {
    "script": "import urllib.request\nimport json\nresp = urllib.request.urlopen('http://api.example.com/data')\ndata = json.loads(resp.read())\nreturn json.dumps(data), metadata, msgType",
    "timeout": "30s"
  }
}
```

### Defining Helper Functions

```json
{
  "id": "s8",
  "type": "x/python",
  "name": "Helper Functions",
  "configuration": {
    "script": "def calculate_factorial(n):\n    if n <= 1:\n        return 1\n    return n * calculate_factorial(n - 1)\nimport json\ndata = json.loads(msg) if isinstance(msg, str) else msg\ndata[\"factorial\"] = calculate_factorial(data.get(\"n\", 5))\nreturn json.dumps(data), metadata, msgType"
  }
}
```

## Limitations and Notes

1. **Subprocess model**: A new Python subprocess is spawned for each message, which has startup overhead. Not suitable for extremely latency-sensitive scenarios
2. **Concurrency control**: Use `maxRunning` to limit the maximum number of concurrent subprocesses to prevent resource exhaustion. Default: 10
3. **Timeout protection**: Always configure `timeout` to prevent infinite loops or blocked network requests from hanging subprocesses
4. **JSON serialization**: Return values are passed through JSON serialization. Python-specific objects (datetime, set, etc.) are not supported and must be converted to basic types first
5. **Process isolation**: Each invocation runs in an isolated process. Python global state (module-level variables) is not preserved between calls
6. **Third-party libraries**: All installed Python packages are available, but they must be installed in the environment
7. **File paths**: File mode `.py` paths must be absolute or correctly relative to the working directory

## Environment Setup

### Linux / macOS

```bash
# Ubuntu/Debian
sudo apt-get install python3

# macOS (Homebrew)
brew install python3
```

### Windows

1. Download the installer from [python.org](https://www.python.org/downloads/)
2. Check **Add Python to PATH** during installation
3. Verify the installation:

```cmd
python --version
# Python 3.x.x
```

### Docker

```dockerfile
FROM golang:1.21
RUN apt-get update && apt-get install -y python3
```

### Environment Variable

You can specify the Python executable path via the `PYTHON_PATH` environment variable:

```bash
export PYTHON_PATH=/usr/bin/python3
```

Or specify it in Go code via the `pythonPath` configuration:

```json
{
  "pythonPath": "/usr/bin/python3"
}
```
