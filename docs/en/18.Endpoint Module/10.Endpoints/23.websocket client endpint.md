***Websocket Client Endpoint*** actively connects to a remote WebSocket server, receives server-pushed messages, and routes them to rule chains for processing. It supports automatic reconnection, heartbeat keep-alive, custom headers, and message type filtering. Ideal for real-time data subscription, WebSocket protocol proxying, and IoT device integration.

## Type

endpoint/ws_client

## Core Features

### 🔄 Auto Reconnection
Automatically attempts to reconnect to the server after disconnection at configured intervals, ensuring connection reliability.

### 💓 Heartbeat Keep-Alive
Supports three heartbeat modes:
- **Default**: Sends WebSocket protocol-level Ping frame
- **Custom Content**: Sends custom heartbeat data via TextMessage
- **Callback Function**: Full control over heartbeat logic via Go API

### 📨 Message Type Filtering
Supports filtering received data by message type:
- **TextMessage**: Text messages
- **BinaryMessage**: Binary messages
- Each type can be independently enabled/disabled for flexible message handling

### 🔗 Custom Headers
Supports custom HTTP headers during connection for authentication and authorization scenarios.

## Configuration

| Field              | Type   | Required | Description                                                    | Default |
|--------------------|--------|----------|----------------------------------------------------------------|---------|
| server             | string | Yes      | WebSocket server URL, e.g., ws://127.0.0.1:8080/ws or wss://  | -       |
| headers            | object | No       | Custom HTTP headers for connection, JSON format                | -       |
| reconnectInterval  | int    | No       | Reconnection interval in seconds, 0 for no reconnection       | 5       |
| heartbeatInterval  | int    | No       | Heartbeat send interval in seconds, 0 for no heartbeat        | 0       |
| heartbeatData      | string | No       | Heartbeat content, empty string sends Ping frame, non-empty sends TextMessage | "" |
| allowText          | bool   | No       | Whether to accept TextMessage                                  | true    |
| allowBinary        | bool   | No       | Whether to accept BinaryMessage                                | true    |

## Complete Configuration Examples

### Basic WebSocket Client
```json
{
  "id": "ws_client",
  "type": "endpoint/ws_client",
  "configuration": {
    "server": "ws://127.0.0.1:8080/ws",
    "reconnectInterval": 5
  },
  "routers": [
    {
      "from": {
        "path": ""
      },
      "to": {
        "path": "chain:processMessage"
      }
    }
  ]
}
```

### Client with Authentication and Heartbeat
```json
{
  "id": "ws_auth_client",
  "type": "endpoint/ws_client",
  "configuration": {
    "server": "wss://api.example.com/v1/stream",
    "headers": {
      "Authorization": "Bearer token123"
    },
    "reconnectInterval": 3,
    "heartbeatInterval": 30,
    "heartbeatData": "{\"type\":\"ping\"}"
  },
  "routers": [
    {
      "from": {
        "path": ""
      },
      "to": {
        "path": "chain:handleStream"
      }
    }
  ]
}
```

### Binary-Only Client
```json
{
  "id": "ws_binary_client",
  "type": "endpoint/ws_client",
  "configuration": {
    "server": "ws://192.168.1.100:9090/binary",
    "allowText": false,
    "allowBinary": true,
    "reconnectInterval": 5,
    "heartbeatInterval": 60
  },
  "routers": [
    {
      "from": {
        "path": ""
      },
      "to": {
        "path": "chain:binaryHandler"
      }
    }
  ]
}
```

## Go Code Example

```go
config := engine.NewConfig()
client := &websocket.WsClient{}
client.Init(config, types.Configuration{
    "server":             "ws://127.0.0.1:8080/ws",
    "reconnectInterval":  5,
    "heartbeatInterval":  30,
    "heartbeatData":      "",
})

router := impl.NewRouter().From("").Process(func(router endpoint.Router, exchange *endpoint.Exchange) bool {
    // Process received messages
    return true
}).End()
client.AddRouter(router)
client.Start()
```
