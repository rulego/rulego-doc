# 结构化日志

`github.com/rulego/streamsql/logger` 包在原有 printf 风格日志之上叠加了一层**结构化日志**：以 key-value 形式携带上下文，支持 logfmt 文本与 JSON 两种输出。

## 概述

printf 日志对人友好，但对日志聚合系统（ELK、Loki、Datadog 等）不友好——要从 `"window fired for device d1 in 5ms"` 里抽字段得写正则。结构化日志把上下文拆成独立的 key-value，聚合系统可以直接按 `device=d1`、`duration=5ms` 建索引、做过滤。

这层是**叠加式（additive）**的：

- `Debug`/`Info`/`Warn`/`Error` 的 printf 签名**完全不变**，已有的自定义 `Logger` 实现和调用点继续工作。
- 新增 `DebugFields`/`InfoFields`/`WarnFields`/`ErrorFields` 四个方法，接收 `...Field`。
- 新增 `NewLoggerWithFormat` 构造器选择输出编码。

::: tip 不破坏现有代码
你既有的 `logger.Logger` 实现（只实现了 printf 四个方法）不需要任何改动。只有想输出结构化 key-value 时，才需要实现 `StructuredLogger` 接口。
:::

## Field 构造器

```go
type Field struct {
    Key   string
    Value interface{}
}
```

| 构造器 | 说明 |
|--------|------|
| `logger.String(k, v)` | 字符串值 |
| `logger.Int(k, v)` | `int` 值 |
| `logger.Int64(k, v)` | `int64` 值 |
| `logger.Float64(k, v)` | `float64` 值 |
| `logger.Bool(k, v)` | `bool` 值 |
| `logger.Err(err)` | 错误值，固定写入键 `"error"`；`nil` 会被保留为 null，不会 panic |
| `logger.Duration(k, d)` | `time.Duration`，以人类可读字符串记录，如 `"12ms"` |
| `logger.Any(k, v)` | 任意值；有对应类型构造器时优先用它 |

::: tip 为什么用类型化构造器
`Field.Value` 是 `interface{}`。直接传 `Field{Key: "n", Value: 42}` 时 `42` 会被装箱为 `int`，但如果在期望 `int64` 的上下文里读它就会类型断言失败。类型化构造器在调用点就钉死类型，避免装箱歧义。
:::

## 两种输出格式

```go
type Format int

const (
    TextFormat Format = iota // "[ts] [LEVEL] msg key=value"，logfmt 风格。默认
    JSONFormat               // 每行一个 JSON 对象
)
```

同一条日志调用，两种格式的输出：

```go
logger.InfoFields("window fired",
    logger.String("device", "d1"),
    logger.Int("count", 42),
    logger.Duration("duration", 5200*time.Microsecond),
)
```

Text（默认）：

```
[2026-07-08 10:00:00.123] [INFO] window fired device=d1 count=42 duration=5.2ms
```

JSON：

```
{"ts":"2026-07-08 10:00:00.123","level":"INFO","msg":"window fired","device":"d1","count":42,"duration":"5.2ms"}
```

两种格式都带固定三字段：`ts`（时间戳，毫秒精度）、`level`、`msg`，后面跟调用方传入的字段。

## 创建结构化日志器

```go
func NewLoggerWithFormat(level Level, output io.Writer, format Format) Logger
```

```go
import (
    "os"

    "github.com/rulego/streamsql/logger"
)

// JSON 格式，INFO 级别，输出到 stdout
log := logger.NewLoggerWithFormat(logger.INFO, os.Stdout, logger.JSONFormat)
```

把它接到 StreamSQL 内部，让所有内置日志都走它：

```go
logger.SetDefault(logger.NewLoggerWithFormat(logger.INFO, os.Stdout, logger.JSONFormat))

ssql := streamsql.New() // 内部日志会走上面的 JSON 日志器
```

::: tip 与 Option 的关系
`streamsql.WithLogLevel(level)` 和 `streamsql.WithDiscardLog()` 仍然可用——结构化日志层是叠加的，不破坏它们原有的行为。要彻底替换输出格式，用 `SetDefault` 装入结构化日志器。
:::

级别常量与 printf 日志一致：`DEBUG`、`INFO`（默认）、`WARN`、`ERROR`、`OFF`。

## StructuredLogger 接口

自定义日志器想输出结构化字段时，实现这个接口：

```go
type StructuredLogger interface {
    DebugFields(msg string, fields ...Field)
    InfoFields(msg string, fields ...Field)
    WarnFields(msg string, fields ...Field)
    ErrorFields(msg string, fields ...Field)
}
```

`logger` 包内置的 `defaultLogger`（`NewLogger` / `NewLoggerWithFormat` 返回的实现）**同时实现了** `Logger` 和 `StructuredLogger`，所以 printf 和结构化两条路径都用同一份编码逻辑。`discardLogger` 也实现了 `StructuredLogger`（四个方法都是空实现）。

## 包级 Helper

```go
func InfoFields(msg string, fields ...Field)
func WarnFields(msg string, fields ...Field)
func ErrorFields(msg string, fields ...Field)
func DebugFields(msg string, fields ...Field)
```

这四个函数操作**全局默认日志器**（`SetDefault` 设置的那个，默认是 `NewLogger(INFO, os.Stdout)`）：

```go
logger.InfoFields("emit", logger.String("device", "d1"), logger.Int("bytes", 128))
```

::: warning printf-only 日志器的回退
如果默认日志器**没有**实现 `StructuredLogger`（比如你塞了一个只实现了 printf 四方法的自定义 `Logger`），`InfoFields` 等包级 helper 会把所有 field 拼成 `"key=value key=value"` 文本、附在消息后走一次 printf 调用，**不会丢上下文**，但也不会得到 JSON 输出。
:::

printf 风格的包级 helper（`logger.Info`、`logger.Warn`、`logger.Error`、`logger.Debug`）、构造器（`NewLogger`、`NewDiscardLogger`）、访问器（`SetDefault`、`GetDefault`）以及 `Logger` 接口本身（`Debug`/`Info`/`Warn`/`Error`/`SetLevel`）全部保留不变。

## 完整示例

```go
package main

import (
    "fmt"
    "os"
    "time"

    "github.com/rulego/streamsql"
    "github.com/rulego/streamsql/logger"
)

func main() {
    // 1. 安装 JSON 结构化日志器作为全局默认
    logger.SetDefault(logger.NewLoggerWithFormat(logger.INFO, os.Stdout, logger.JSONFormat))

    ssql := streamsql.New()
    defer ssql.Stop()

    ssql.Execute(`SELECT deviceId, AVG(temperature) AS avg_t
                  FROM stream
                  GROUP BY deviceId, TumblingWindow('5s')`)
    ssql.AddSink(func(rows []map[string]interface{}) {
        if len(rows) > 0 {
            // 在 sink 里打一条结构化日志
            logger.InfoFields("window emitted",
                logger.String("device", fmt.Sprintf("%v", rows[0]["deviceId"])),
                logger.Any("avg_t", rows[0]["avg_t"]),
            )
        }
    })

    for i := 0; i < 3; i++ {
        ssql.Emit(map[string]interface{}{
            "deviceId":    "d1",
            "temperature": 20.0 + float64(i),
        })
    }

    time.Sleep(6 * time.Second)
}
```

运行后，stdout 会出现形如（StreamSQL 内部的窗口/调度日志也会是同样格式）：

```
{"ts":"2026-07-08 10:00:05.123","level":"INFO","msg":"window emitted","device":"d1","avg_t":21}
```
