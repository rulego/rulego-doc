# 流表 JOIN 元数据增强

## 业务场景

设备上报的读数流里只有 `deviceId` 和测量值，没有设备的位置、型号等属性。这些属性相对稳定，存在一张"元数据表"里。要求：每条读数到达时，用 `deviceId` 关联元数据表，把位置/型号补到读数上一并输出（流表 JOIN，stream-table join）。

## SQL

```sql
SELECT deviceId,
       m.location,
       m.model,
       temperature
FROM stream JOIN meta m ON deviceId = m.deviceId
```

## 输入

设备读数流（`stream`）：

```json
{"deviceId": "d1", "temperature": 31.0}
{"deviceId": "d2", "temperature": 27.5}
{"deviceId": "d9", "temperature": 40.0}
```

元数据表（`meta`，用 `RegisterTable("meta", rows)` 注册）：

```json
{"deviceId": "d1", "location": "plantA", "model": "TX-100"}
{"deviceId": "d2", "location": "plantB", "model": "TX-200"}
```

## 输出

```json
{"deviceId": "d1", "location": "plantA", "model": "TX-100", "temperature": 31.0}
{"deviceId": "d2", "location": "plantB", "model": "TX-200", "temperature": 27.5}
```

`d9` 在元数据表里没有匹配，默认 `INNER JOIN` 被丢弃。

## 行为说明

- **默认 INNER JOIN**：元数据表无匹配的读数被丢弃（如 `d9`）。需要保留无匹配读数时用 `LEFT JOIN`，未命中的表字段为 `nil`。
- **JOIN 在 WHERE 之前**执行，所以 `WHERE`、`GROUP BY`、聚合都能引用 JOIN 进来的表字段（如 `GROUP BY m.location`）。
- **`RegisterTable` 必须在 `Execute` 之后调用**：先 `Execute(sql)`，再 `ssql.RegisterTable("meta", []map[string]any{...})`，否则 JOIN 找不到表、全部读数被丢弃。
- 表别名 `m`，匹配列写 `ON deviceId = m.deviceId`（**流字段不加别名**，表字段用 `m.xxx`）。
- 元数据表是"静态维度表"，适合设备/用户/产品属性这类低频变更数据；高频变更维表可在节点侧定时刷新。

::: tip 如何运行
JOIN 查询是非聚合查询，用 `EmitSync` 同步拿结果；注意 `Execute` 之后要用 `RegisterTable` 注册元数据表。完整样板见[如何运行案例 SQL](./00.概述.md#如何运行案例-sql)。
:::
