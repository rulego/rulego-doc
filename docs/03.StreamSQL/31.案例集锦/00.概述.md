# StreamSQL 案例集锦

本章提供了StreamSQL在各种实际业务场景中的应用案例，涵盖数据合并、变更数据捕获、实时分析等常见的流处理需求。所有案例都基于StreamSQL项目的实际源码，确保可运行性和实用性。

## 案例分类

所有案例均为 IoT 场景、SQL 已实跑验证，并提供输入/输出。运行方式见下文[如何运行案例 SQL](#如何运行案例-sql)。

### 🔧 数据处理
- [数据过滤与转换](/pages/streamsql-case-filter/) - 非聚合：过滤、算术换算、CASE 分级（`EmitSync`）
- [流表 JOIN 元数据增强](/pages/streamsql-case-join/) - 用维度表给设备读数补位置/型号等属性

### ⏱️ 窗口聚合
- [滑动窗口与持续检测](/pages/streamsql-case-sliding/) - `SlidingWindow` + `HAVING` 做"持续 N 秒超阈值"
- [会话窗口与设备在线](/pages/streamsql-case-session/) - `SessionWindow` 按活动间隙切分在线会话
- [IoT 温度告警与指标聚合](/pages/streamsql-case-iot-temperature/) - 单点告警 / 滚动 / 计数 / 触发 / 全局窗口模式总览

## 如何运行案例 SQL

案例里只给 SQL 和输入输出；要真正跑起来，用下面这段统一的 Go 样板——把案例 SQL 粘进 `sql` 变量、按案例给的输入发数据即可。

```bash
go get github.com/rulego/streamsql
```

```go
package main

import (
	"fmt"
	"time"

	"github.com/rulego/streamsql"
)

func main() {
	ssql := streamsql.New()
	defer ssql.Stop()

	// ① 把案例里的 SQL 粘贴到这里
	sql := `SELECT deviceId, temperature FROM stream WHERE temperature > 30`
	if err := ssql.Execute(sql); err != nil {
		panic(err)
	}

	// ② 二选一：看案例属于"非聚合"还是"聚合/窗口"

	// 方式 A：非聚合查询（无 GROUP BY/窗口）用 EmitSync，同步拿单条结果
	// result, err := ssql.EmitSync(map[string]any{"deviceId": "d1", "temperature": 32.0})
	// fmt.Println(result)

	// 方式 B：聚合 / 窗口查询用 Emit + AddSink，窗口关闭时按批回调
	ssql.AddSink(func(batch []map[string]any) {
		for _, row := range batch {
			fmt.Printf("%+v\n", row)
		}
	})
	ssql.Emit(map[string]any{"deviceId": "d1", "temperature": 32.0})

	time.Sleep(500 * time.Millisecond) // 等异步 sink 落盘
}
```

**按查询类型选 API：**

| 查询类型 | 用什么 | 说明 |
|---------|--------|------|
| 非聚合（过滤/转换/分析函数） | `EmitSync(data)` | 同步返回单条结果；不命中 WHERE 返回 `nil` |
| 聚合 / 窗口 / 全局窗口 | `Emit(data)` + `AddSink(...)` | 窗口关闭时按批回调 |

::: tip 测试时间窗口免 sleep
写测试或想立刻看到时间窗口的输出时，用 `ssql.TriggerWindow()` 立即冲窗口，不必 `time.Sleep` 等窗口到期。详见 [IoT 温度告警案例 · 场景 D](./06.IoT温度告警与指标聚合.md#场景-d-加餐-用-triggerwindow-写确定性测试)。
:::
