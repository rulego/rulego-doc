规则链是 RuleGo 的核心概念，由多个组件节点和连接关系组成，定义了数据流转和处理规则。RuleGo-Server 提供完整的 REST API 进行规则链的生命周期管理。

## 列出规则链

```http
GET /api/v1/rules?keywords=&root=&disabled=&category=&page=1&size=20
Authorization: Bearer {token}
```

查询参数：

| 参数 | 类型 | 说明 |
|------|------|------|
| `keywords` | string | 关键词过滤 |
| `root` | bool | 过滤根规则链 |
| `disabled` | bool | 过滤已禁用的规则链 |
| `category` | string | 按分类过滤 |
| `page` | int | 页码 |
| `size` | int | 每页数量 |

## 获取规则链

```http
GET /api/v1/rules/{id}
Authorization: Bearer {token}
```

返回规则链的完整 DSL JSON。

## 获取最新规则链

```http
GET /api/v1/rules/_/latest
Authorization: Bearer {token}
```

返回当前用户最近修改的规则链。路径中的 ID 参数会被忽略。

## 创建/更新规则链

```http
POST /api/v1/rules/{id}
Authorization: Bearer {token}
Content-Type: application/json

{规则链 DSL JSON}
```

创建或更新规则链，保存后自动加载到引擎中。

## 更新规则链基础信息

```http
POST /api/v1/rules/{id}/base
Authorization: Bearer {token}
Content-Type: application/json

{
  "name": "新名称",
  "additionalInfo": {
    "description": "新描述",
    "category": "数据处理"
  }
}
```

仅更新规则链的元数据，不影响节点和连接。

## 保存规则链配置变量

```http
POST /api/v1/rules/{id}/config/{varType}
Authorization: Bearer {token}
Content-Type: application/json

{变量配置 JSON}
```

## 删除规则链

```http
DELETE /api/v1/rules/{id}
Authorization: Bearer {token}
```

## 部署操作

```http
POST /api/v1/rules/{id}/operate/{operateType}
Authorization: Bearer {token}
```

| operateType | 说明 |
|-------------|------|
| `start` | 部署规则链（开始运行） |
| `stop` | 停止规则链（取消部署） |
| `set-to-main` | 设为主规则链 |

## 规则链 DSL 结构

```json
{
  "ruleChain": {
    "id": "myChain",
    "name": "我的规则链",
    "debugMode": false,
    "root": false,
    "disabled": false,
    "additionalInfo": {
      "description": "规则链描述",
      "category": "分类",
      "inputSchema": {},
      "createTime": "2026/01/01 00:00:00"
    }
  },
  "metadata": {
    "endpoints": [],
    "nodes": [
      {
        "id": "node_1",
        "type": "jsFilter",
        "name": "过滤节点",
        "debugMode": false,
        "configuration": {
          "jsScript": "return msg.age > 18;"
        },
        "additionalInfo": {
          "layoutX": 300,
          "layoutY": 200
        }
      }
    ],
    "connections": [
      {
        "fromId": "node_1",
        "toId": "node_2",
        "type": "True"
      }
    ]
  }
}
```

### additionalInfo 字段说明

| 字段 | 说明 |
|------|------|
| `description` | 规则链描述 |
| `category` | 分类标识 |
| `inputSchema` | 输入参数 JsonSchema（用于 MCP 工具、AI Agent） |
| `createTime` / `updateTime` | 创建/更新时间 |
| `layoutX` / `layoutY` | 编辑器画布坐标（仅节点级别） |
| `username` | 所属用户

## 应用集成端点

保存规则链后，系统自动为其生成以下端点：

| 端点 | 说明 |
|------|------|
| `POST /api/v1/rules/{id}/execute/{msgType}` | 同步执行 |
| `POST /api/v1/rules/{id}/notify/{msgType}` | 异步通知 |
| `POST /api/v1/rules/{id}/v1/chat/completions` | OpenAI 兼容对话端点 |
| `/api/v1/rules/{id}/mcp/sse` | MCP SSE 端点 |
