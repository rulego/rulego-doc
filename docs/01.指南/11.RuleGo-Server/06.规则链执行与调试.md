RuleGo-Server 提供同步执行、异步通知、WebSocket 实时调试等多种规则链执行方式。

## 同步执行

等待规则链执行完成后返回结果：

```http
POST /api/v1/rules/{id}/execute/{msgType}
Authorization: Bearer {token}
Content-Type: application/json

{
  "message": "hello",
  "metadata": {
    "key": "value"
  }
}
```

- `{id}`：规则链 ID
- `{msgType}`：消息类型，自定义字符串

响应为规则链处理后的输出数据。

### curl 示例

```shell
curl -X POST http://localhost:9090/api/v1/rules/myChain/execute/TEST \
  -H "Content-Type: application/json" \
  -d '{"temperature": 36.5, "deviceId": "sensor001"}'
```

## 异步通知

触发规则链执行后立即返回，不等待结果：

```http
POST /api/v1/rules/{id}/notify/{msgType}
Authorization: Bearer {token}
Content-Type: application/json

{
  "message": "hello"
}
```

适用于日志处理、消息分发等不需要即时结果的场景。

## WebSocket 实时调试

通过 WebSocket 连接实时查看规则链执行过程中每个节点的输入/输出数据。

### 连接

```
ws://{host}:{port}/api/v1/logs/ws/{chainId}/{clientId}?token={jwtToken}
```

参数说明：

| 参数 | 说明 |
|------|------|
| `chainId` | 规则链 ID |
| `clientId` | 客户端唯一标识（自定义） |
| `token` | JWT 令牌（query 参数） |

### 消息格式

服务端推送的调试数据包含：
- **节点 ID**：当前执行的节点
- **方向**：`IN`（输入）/ `OUT`（输出）
- **消息数据**：节点的输入/输出内容
- **元数据**：上下文元数据
- **消息类型**：msgType

### 前端连接示例

```javascript
const ws = new WebSocket(
  'ws://localhost:9090/api/v1/logs/ws/myChain/client001?token=' + token
)
ws.onmessage = (event) => {
  const data = JSON.parse(event.data)
  console.log(`[${data.direction}] Node: ${data.nodeId}`, data.msg)
}
```

连接支持自动重连（指数退避，最多 10 次）。

## 节点调试数据

通过 HTTP 接口获取节点的历史调试日志：

```http
GET /api/v1/logs/debug?chainId={chainId}&nodeId={nodeId}
Authorization: Bearer {token}
```

## 执行历史

如果开启了 `save_run_log = true`，系统会记录每次规则链执行的历史。

### 查询执行历史

```http
GET /api/v1/logs/runs?chainId={chainId}&page=1&size=20&startTime=&endTime=
Authorization: Bearer {token}
```

### 获取单条执行记录

```http
GET /api/v1/logs/runs?id={runId}
Authorization: Bearer {token}
```

### 删除执行记录

```http
DELETE /api/v1/logs/runs?chainId={chainId}
DELETE /api/v1/logs/runs?id={runId}
Authorization: Bearer {token}
```

## 在编辑器中调试

RuleGo-Editor 提供了可视化的调试功能：

1. 打开规则链，点击工具栏【运行】按钮
2. 配置输入消息（msgType、请求体、元数据）
3. 可选择从特定节点开始执行
4. 执行过程中，画布上节点会以颜色标识状态：
   - **蓝色**：正在处理
   - **绿色**：执行成功
   - **红色**：执行失败
5. 底部调试控制台实时显示数据流经每个节点的 IN/OUT 数据

## OpenAI 兼容执行

如果规则链包含 `ai/agent` 节点，可通过 OpenAI 兼容接口执行：

```http
POST /api/v1/rules/{id}/v1/chat/completions
Authorization: Bearer {token}
Content-Type: application/json

{
  "model": "rulego",
  "messages": [
    {"role": "user", "content": "你好"}
  ],
  "stream": true
}
```

支持 SSE 流式输出，与 OpenAI API 格式完全兼容。
